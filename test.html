1. 三个操作的概念区分

操作 按钮位置 行为 存储位置
暂存 toolbar 最右侧（已有） 临时保存当前编辑内容，防崩溃 当前临时状态（不变）
保存草稿 toolbar 最右侧（新增） 写入草稿存储，可多份草稿 IndexedDB / ~/Documents/R-Markdown/drafts/
定稿 toolbar 最右侧（新增） 导出 .md 文件到磁盘指定目录，可选删除草稿 用户指定的磁盘目录

草稿列表入口：左侧窄侧边栏，仅放置一个草稿列表按钮，点击弹出 DraftListDialog。

4. 交互流程

4.1 标题提取策略（用于预填弹窗，用户可编辑）

提取优先级（命中即止），提取到则预填到弹窗，未提取到则字段为空需手动输入：

优先级 来源 正则
1 <title> 组件 body /^<title\b[^>]*>([\s\S]*?)<\/title>/m
2 Front-matter title 字段 `/^---\s\n(?:[\s\S]?\n)?title:\s*(.+?)(?:\n\ $)/m`
3 Markdown # 一级标题 /^#\s+(.+)/m

不提取 <p-title> 段落标题（仅为文内小节标题，不作为文章主标题）。

4.2 保存草稿

点击「保存草稿」按钮  ↓弹出「保存草稿」BaseDialog  ┌──────────────────────────────┐  │  保存草稿                     │  │                              │  │  草稿标题：                   │  │  ┌──────────────────────────┐ │  │  │ 功能全集：排版组件指南    │ │  ← 可编辑，已提取的标题预填  │  └──────────────────────────┘ │  │                              │  │            [取消]    [保存]   │  └──────────────────────────────┘  ↓ 点击「保存」  ↓标题为空？  ├─ 是 → 弹窗内部红色提示「请输入标题」，不关闭弹窗  └─ 否 → 继续  ↓如果当前编辑器已关联草稿 ID → 覆盖更新该草稿（保持原 ID）如果新文章 → 新建草稿记录，写入 DraftStorage  ↓关闭弹窗  ↓左侧栏草稿角标刷新  ↓Toast「草稿已保存」  ↓编辑器关联该草稿 ID

关键约束：

标题必填，为空时无法保存，按钮置灰或点击后红字提示
如果当前编辑器和已有草稿内容完全相同 → Toast「内容无变化，无需重复保存」，不写入

4.2 加载草稿

点击侧边栏草稿项  ↓当前编辑器有未保存内容？  ├─ 是 → BaseDialog 确认弹窗  │        「当前内容尚未保存，是否保存后再打开？」  │        [保存并打开] [不保存直接打开] [取消]  └─ 否 → 直接加载  ↓编辑器关联该草稿 ID  ↓侧边栏圆点标记移到新草稿

4.3 定稿

点击「定稿」按钮  ↓弹出「定稿」BaseDialog  ┌──────────────────────────────┐  │  定稿                         │  │                              │  │  文章标题：                   │  │  ┌──────────────────────────┐ │  │  │ 功能全集：排版组件指南    │ │  ← 可修改  │  └──────────────────────────┘ │  │                              │  │  保存位置：                   │  │  ┌──────────────────────────┐ │  │  │ ~/Documents/公众号文章/   │ │  ← 路径选择器  │  │                    [浏览]│ │  │  └──────────────────────────┘ │  │                              │  │  文件名：功能全集：排版组件指南.md │ ← 只读预览  │                              │  │            [取消]    [保存]   │  └──────────────────────────────┘  ↓ 点击「保存」  ↓检查目标路径是否已存在同名文件？  ├─ 是 → 弹出覆盖确认弹窗  │   「{文件名}.md 已存在，是否覆盖？」  │   [取消] [覆盖]  │   ├─ 取消 → 回到定稿弹窗  │   └─ 覆盖 → 继续写入  └─ 否 → 继续写入  ↓将 markdown 内容写入 `{路径}/{标题}.md`  ↓写入成功 → Toast「已保存到 {路径}」  ↓弹出确认弹窗：  「定稿已保存，是否删除该草稿？」  [是，删除草稿] [保留草稿]  ↓  ├─ 删除 → 从 DraftStorage 删除 + 侧边栏移除 + 取消编辑器草稿关联  └─ 保留 → 不做额外操作

标题处理：

默认按标题提取策略（4.1）提取
用户可手动修改
自动去除文件名非法字符（/ \ : * ? " < > |）

路径选择：

桌面端（Tauri）：使用 Tauri 原生目录选择对话框
Web 端：浏览器不支持选择目录，降级为普通下载（浏览器默认下载路径）
记忆上次选择的目录

5. 数据模型

interface Draft {  id: string            // 唯一 ID，时间戳生成  title: string         // 从 markdown 首行 # 标题提取  content: string       // 完整 markdown 原文  preview: string       // 正文前 80 字（去标记），用于侧边栏展示  createdAt: number     // 首次保存时间  updatedAt: number     // 最后保存时间}

8. 关键决策

1.草稿列表用弹窗而非侧边栏内嵌：左侧仅保留一个图标按钮，点击弹出 DraftListDialog。符合已有 BaseDialog 统一风格，不挤占编辑区域。
1.三种按钮独立：暂存（临时恢复）、保存草稿（持久化版本）、定稿（导出成品）职责清晰
1.定稿路径选择：桌面端用原生对话框，Web 端降级为下载
1.内容去重：保存草稿时检查内容是否与上次完全相同，避免重复条目
1.主题色一致性：弹窗按钮、角标、卡片边框均使用 var(--accent)
